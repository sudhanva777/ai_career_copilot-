# AI Career Copilot

A full-stack AI-powered career platform for job seekers. Upload a resume, get an instant ATS score and skill gap analysis, match against job descriptions, practice mock interviews with an LLM coach, and export a polished PDF career report — all running locally with no cloud AI costs.

---

## Features

| # | Feature | Description |
|---|---------|-------------|
| 1 | **Secure auth** | Register / login with bcrypt-hashed passwords. JWT stored in an HttpOnly, SameSite=lax cookie — never exposed to JavaScript. |
| 2 | **Resume upload & analysis** | PDF and DOCX accepted (max 5 MB). Magic-bytes validation. Semantic skill extraction via sentence embeddings (cosine similarity ≥ 0.35). Outputs ATS score (0–100), detected skills with confidence, top role matches, and gap skills. |
| 3 | **AI rewrite suggestions** | Ollama generates concrete, copy-paste-ready bullet-point improvements for every skill gap. Falls back to template suggestions when Ollama is offline. Suggestions are persisted so the page reloads instantly. |
| 4 | **Score history** | Every resume version tracked. Line chart shows ATS score progression across uploads with delta indicators and new-skill callouts. |
| 5 | **Job description matching** | Paste text or a URL. The backend fetches and parses the page (BeautifulSoup), extracts JD skills, compares against your resume, and returns a match score, matched skills, missing skills, and ready-to-paste bullet points for each gap. |
| 6 | **AI interview coach** | Role-specific questions generated by Llama 3 with 5 categories (Technical / Behavioral / Situational / Problem Solving / Communication). Each answer is scored 0–10 with LLM feedback and an ideal answer. Practice Mode lets you re-attempt without restrictions. |
| 7 | **Interview summary & stats** | Per-session RadarChart and category score bars. Streak tracking, weekly session count, and rolling average score on the dashboard. |
| 8 | **Career Health dashboard** | Composite Career Health gauge (ATS 40%, interview avg 35%, best job match 25%). Activity feed, recommended next steps, and onboarding wizard for first-time users. |
| 9 | **Skill confidence visualisation** | Sorted confidence-bar breakdown of every detected skill across three tiers (Strong / Good / Partial) in the Analysis page. |
| 10 | **PDF career report** | Print-safe white-theme report page at `/report/:resumeId` with all scores, role predictions, skill breakdown, and gap list. `window.print()` exports to PDF. |

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React 18, Vite, React Router v6, vanilla CSS utility classes |
| Charts | Recharts (LineChart, RadarChart) |
| Backend | FastAPI, SQLAlchemy 2, Alembic |
| Database | SQLite (dev) / PostgreSQL (prod) |
| NLP | spaCy `en_core_web_sm`, Sentence-Transformers `all-MiniLM-L6-v2` |
| LLM | Ollama (Llama 3) via local HTTP |
| Auth | JWT in HttpOnly cookie (`session`), passlib + bcrypt 3.2.x |
| Rate limiting | SlowAPI (always enabled; 3 req/min register, 5 req/min login) |
| Infrastructure | Docker Compose, Nginx |

## Setup

### Docker (recommended)

**Prerequisites:** Docker Desktop with Compose v2; at least 4 GB of free RAM for the Ollama model.

```bash
# 1. Copy the env template
cp .env.example .env

# 2. Generate a secure SECRET_KEY (must be at least 32 characters)
python -c "import secrets; print(secrets.token_hex(32))"
# Paste the output as SECRET_KEY= in .env

# 3. Start all four services (db, backend, frontend, ollama)
docker compose up --build

# 4. Pull the Llama 3 model — first run only, takes a few minutes
bash scripts/setup_ollama.sh
```

| Service | URL |
|---------|-----|
| Frontend | http://localhost:3000 |
| Backend API | http://localhost:8000 |
| Interactive API docs | http://localhost:8000/docs |
| Health check | http://localhost:8000/health |

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `SECRET_KEY` | **Yes** | — | JWT signing key. Must be ≥ 32 characters. Server refuses to start otherwise. |
| `DATABASE_URL` | No | `sqlite:///./app.db` | SQLAlchemy connection string. PostgreSQL: `postgresql://user:pass@host/dbname` |
| `ALLOWED_ORIGINS` | No | `http://localhost:3000,http://localhost:5173` | Comma-separated CORS origins |
| `OLLAMA_BASE_URL` | No | `http://localhost:11434` | Ollama endpoint. In Docker Compose: `http://ollama:11434` |
| `UPLOAD_DIR` | No | `./uploads` | Root for uploaded files. `uploads/resumes/` is created automatically on startup. |
| `ENVIRONMENT` | No | `production` | Set to `development` for plain-HTTP (non-Secure) cookies. |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | No | `1440` | JWT lifetime in minutes (24 hours, matches cookie `max_age`). |

## API Reference

### Auth — `/api/v1/auth`

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/register` | No | Create account. Body: `{name, email, password}`. Rate-limited: 3 req/min. |
| `POST` | `/login` | No | Log in, receive `session` cookie. Body: `{email, password}`. Rate-limited: 5 req/min. |
| `GET` | `/me` | Yes | Return the authenticated user. |
| `POST` | `/logout` | Yes | Clear the `session` cookie. |

### Resume — `/api/v1/resume`

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/upload` | Yes | Upload PDF or DOCX (max 5 MB, multipart). Returns `{resume_id, filename, status}`. |
| `GET` | `/list` | Yes | List all resumes for the current user, newest first. |
| `DELETE` | `/{resume_id}` | Yes | Delete a resume and its analysis. |
| `GET` | `/analysis/{resume_id}` | Yes | Full analysis: ATS score, skills, roles, gaps. |
| `GET` | `/history` | Yes | All resume versions oldest→newest with ATS score, delta, and version label. |
| `POST` | `/rewrite/{resume_id}` | Yes | Generate and persist AI rewrite suggestions (falls back to templates if Ollama offline). |
| `GET` | `/rewrite/{resume_id}` | Yes | Return previously generated suggestions. 404 if not yet generated. |

### Interview — `/api/v1/interview`

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/start` | Yes | Start session. Body: `{analysis_id, target_role, practice_mode}`. Returns `{session_id, questions, ai_available, practice_mode}`. |
| `POST` | `/answer` | Yes | Submit answer. Body: `{session_id, question_id, answer}`. Returns `{score, feedback, ideal_answer, session_complete, overall_score}`. |
| `GET` | `/session/{session_id}/summary` | Yes | Full session breakdown with per-category scores and Q&A list. |
| `GET` | `/stats` | Yes | `{sessions_this_week, avg_score, streak_days, total_sessions}`. |

### Jobs — `/api/v1/jobs`

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/match` | Yes | Match resume against JD. Body: `{resume_id, jd_text?}` or `{resume_id, jd_url?}`. Returns match score, matched skills, missing skills. |
| `GET` | `/matches` | Yes | All past JD matches for the current user, newest first. |

### Dashboard — `/api/v1/dashboard`

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `GET` | `/summary` | Yes | Career Health score, ATS/interview/match stats, recent activity feed, next steps. |

### Health

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Deep check: DB, Ollama, NLP. Returns `{"status": "ok"\|"degraded", "db": …, "ollama": …, "nlp": …}` |

## Running Tests

```bash
cd backend
pytest tests/test_integration.py -v
```

The suite uses an in-memory SQLite database, mocks Ollama with `AsyncMock`, mocks ML model loading with `pytest_configure`, and disables the rate limiter via `limiter.enabled = False`. No running services required. 17 test scenarios: health check, full auth flow, resume upload and validation, file-type attacks, full interview flow, duplicate-answer rejection, practice mode, wrong-session rejection, and unauthenticated access.

## Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `RuntimeError: SECRET_KEY must be at least 32 characters` | Key too short or missing | Generate: `python -c "import secrets; print(secrets.token_hex(32))"` |
| `401` on every request locally | `ENVIRONMENT` defaults to `production`, enabling `Secure` cookies that browsers drop over HTTP | Set `ENVIRONMENT=development` in `backend/.env` |
| Cookies not sent after 24 hours | JWT expired | Ensure `ACCESS_TOKEN_EXPIRE_MINUTES=1440` in `.env` |
| Rewrite suggestions GET returns 404 after generating | `rewrite_suggestions_json` is `[]` (no gap skills); old guard used `not []` | Fixed: guard is now `is None` — empty list returns 200 with 0 suggestions |
| Skills rendered as `[object Object]` | Frontend used `skill` (object) instead of `skill.skill` | Fixed: all pages use `skill.skill` for the label |
| Resume analysis poll never resolves | Poll checked `res.stats` (field does not exist) | Fixed: poll checks `res.analysis_id !== undefined` |
| `OperationalError: duplicate column` on `alembic upgrade head` | Old init migration used `Base.metadata.create_all()` (snapshots full schema) | Fixed: all 4 migrations are idempotent via `sa_inspect` checks |
| `ValueError: password longer than 72 bytes` in tests | `bcrypt ≥ 4.0` is incompatible with `passlib 1.7.4` | Fixed: `bcrypt==3.2.2` pinned in `requirements.txt` |
| Ollama questions not generating | Llama 3 model not pulled or Ollama not running | Run `bash scripts/setup_ollama.sh`; check `docker compose logs ollama` |
| File upload rejected despite valid PDF | Wrong magic bytes (renamed extension) | Upload a real PDF/DOCX — server checks file content, not just the extension |
